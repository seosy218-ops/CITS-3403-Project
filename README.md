# IDEA 1: TuneSpace

**TuneSpace** is a modern music discovery and beat marketplace platform that combines the discovery experience of short-form social media with the functionality of a professional beat marketplace.

The platform allows **producers to upload and sell beats**, while **artists and listeners discover music through an algorithm-driven scrolling feed**.

Instead of browsing long lists of beats, TuneSpace enables **instant discovery through a TikTok-style vertical scrolling interface**.

---

# Project Overview

TuneSpace aims to solve a major issue in existing beat marketplaces:

> Finding high-quality beats quickly is difficult.

Current platforms rely on long lists and manual filters. TuneSpace instead uses **algorithmic discovery and social interaction** to surface relevant beats instantly.

The platform combines three core ideas:

| Concept | Description |
|------|------|
| Short-form discovery | Scrollable beat feed similar to social media |
| Music streaming | High quality playback similar to Spotify |
| Beat marketplace | Producers can list and sell beats |

# Core Features

## Beat Discovery Feed

The main interface is a **vertical scrolling feed**.

When the user scrolls:

1. The next beat begins playing automatically
2. Beat metadata appears on screen
3. Social engagement metrics appear in a corner panel

Displayed elements include:

- Beat title
- Producer name
- BPM
- Key
- Genre
- Price
- Likes and engagement

This creates a **fast discovery experience**.

---

# Social Features

TuneSpace integrates lightweight social features:

- ❤️ Like beats
- 🔁 Repost beats
- ➕ Follow producers
- 💬 Comment
- 👂 Listener count

These interactions contribute to the **recommendation algorithm**.

---

# Recommendation Algorithm

Each user's feed is personalized.

The algorithm considers:

- Beats liked
- Genres listened to
- Listening duration
- Replay frequency
- Producers followed
- Purchased beats

This allows the system to **learn a user's musical preferences** and recommend similar content.

Users can also switch genres easily without leaving the feed.

---

# Beat Metadata

Each beat must include important production information.

Required metadata:

- Price
- BPM
- Musical key
- Genre
- Producer name
- Tags

### Tag System

Producers can attach multiple tags such as:

- Trap
- Boom bap
- Jersey club
- Drill
- Rage
- Hyperpop
- Ambient
- Experimental

Tags help beats reach the **correct audience**.

---

# Beat Purchase System

The purchasing system must be simple and secure.

Key requirements:

- Prevent accidental double sales for exclusive licenses
- Display licensing terms clearly
- Support multiple license types

Example license types:

| License Type | Description |
|------|------|
| Basic License | Beat can be sold multiple times |
| Premium License | Higher quality files and extended usage |
| Exclusive License | Beat is removed after purchase |

Once a beat is sold exclusively:

- It is removed from the discovery feed
- It appears in a **successful beats showcase**

---

# Successful Beat Tracking

When a beat is purchased, statistics are displayed:

- Purchase price
- Total plays before purchase
- Total plays after purchase
- Producer attribution

This creates a **reputation system for producers**.

---

### Profile includes

- Producer bio
- Social links
- Beat catalogue

### Profile sections

- Trending beats
- New releases

### External links

Producers can link to:

- Spotify
- SoundCloud
- YouTube

---

# Messaging System

Artists can contact producers directly through a messaging system.

To prevent spam:

1. An **AI chatbot filter** processes incoming messages
2. The chatbot identifies message intent

Example message types:

- Collaboration request
- Custom beat request
- Licensing questions

Legitimate messages are forwarded to the producer.

---

# Audio Visualisation

Each beat playback screen includes a **reactive waveform visualizer**.

The waveform responds to:

- Frequency (Hz)
- Volume (dB)

The design should remain **minimal and clean**, focusing primarily on the audio experience.

---

# Copyright Protection

Audio must be protected from misuse.

Possible protections include:

- Encrypted audio streaming
- Disabled direct downloads
- Audio watermarking
- Copyright warnings

These measures reduce unauthorized distribution.

# Monetization (Future Development)

Possible revenue models include:

### Listing Fee
Producers pay a small subscription fee to upload beats.

### Marketplace Commission
A percentage of each beat sale goes to TuneSpace.

### Hybrid Model
Small listing fee combined with reduced commission.

This is **not the immediate focus of development**.

TuneSpace improves this by introducing:
